# 오버라이딩 
## 역할
- 원칙적으로 서브 클래스에서 수퍼 클래스에 있는 같은 이름과 형태로 된 함수를 선언할 수 없지만 오버라이딩을 통해 이를 가능하도록 해준다.

## 기본 형태
- 수퍼 클래스에서 open이 붙은 함수는 서브 클래스에서 override를 붙여 재구현 하면 됨
    - 수퍼 클래스: open fun 함수명() {}
    - 서브 클래스: override fun 함수명() {}

```kotlin
open class Animal(){ 
    open fun eat(){
        println("음식을 먹습니다.")
    }
}

class Cat : Animal(){
    override fun eat(){ 
        println("생선을 먹습니다.")
    }
}

fun main(){
  var t = Cat()
  t.eat()
}

```
